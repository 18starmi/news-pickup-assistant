# AGENTS.md

## 目的
このリポジトリは、特定サイトを巡回し、重要情報だけを抽出・要約・提案する24時間常駐エージェントを構築するためのものです。

## 最優先方針
- セキュリティを優先する
- 未信頼な外部入力を本丸に直接渡さない
- 実行権限と解析権限を分離する
- 実装は小さく始め、差し替え可能にする
- 1ファイルに責務を詰め込みすぎない
- 応答は必ず日本語ですること

## 技術方針
- 言語は Python を採用する
- API は FastAPI を採用する
- DB は最初は SQLite、将来は Postgres へ移行可能な構造にする
- LLM は Provider パターンで抽象化する
- デフォルト実装は OllamaProvider とする
- 収集は Crawl4AI を利用する
- バリデーションは Pydantic を用いる
- テストは pytest を用いる

## セキュリティ方針
- 生HTMLを本丸へ渡さない
- 生HTMLを UI に直接表示しない
- 判定層にシェル実行権限を与えない
- 判定層に通知送信権限を与えない
- APIキーや秘密情報は収集層・通知層など必要最小限に限定する
- Docker Compose では役割ごとにサービスを分離する
- DB 保存前に schema validation を必ず通す

## 作業ルール
- 実装前に PROJECT.md を確認する
- 重要な設計判断は PROJECT.md に追記する
- 大きな変更は小さなコミット単位に分ける
- まず最小実装を作り、その後に拡張する
- 不明点があっても、まず合理的な仮定で前進する
- 仮定した内容は PROJECT.md に残す

## コーディング規約
- 型ヒントを付ける
- dataclass または Pydantic model を適切に使う
- 例外を握りつぶさない
- logging を入れる
- マジックナンバーを避ける
- I/O とドメインロジックを分離する

## ディレクトリ方針
- app/ に本体コード
- tests/ にテスト
- docs/ に設計文書
- infra/ に Docker・環境関連
- data/ はローカル開発用の永続領域
- skills/ は Codex の補助知識

## 実装優先順位
1. ドメインモデル
2. DB スキーマ
3. Provider 抽象
4. Crawl4AI 収集
5. 要約・分類・重要度判定
6. API
7. UI
8. スケジューラ
9. フィードバック学習
10. 通知

## 禁止事項
- 単一巨大ファイル化
- UI から生HTMLを直接扱うこと
- 判定層から直接外部通知すること
- LLM出力を無検証でDBへ保存すること
- 将来の差し替えを困難にする密結合
