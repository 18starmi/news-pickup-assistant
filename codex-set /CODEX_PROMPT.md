# Codex 初回指示

あなたは、このリポジトリで「特定サイトを巡回し、重要情報だけを抽出・要約・提案する24時間常駐エージェント」を実装する。

## 目的
指定したニュースサイト・技術サイト・GitHub・Reddit等を定期巡回し、記事や投稿を収集する。
その後、抽出・整形・重複排除・要約・分類・重要度判定を行い、ユーザーにとって価値が高いものだけを提示する。
ユーザーは提示結果に対して次の3種類の反応を返す:
- 深掘りたい
- 役に立った
- 興味ない
この反応履歴を保存し、今後の選別に反映する。

## 重要な設計思想
- 外部サイトはすべて未信頼入力として扱う
- 生HTMLや未加工本文を本丸に直接渡さない
- 収集、抽出、判定、提示、通知を責務分離する
- Docker Compose 前提で多層防御の構成にする
- 判定LLMにはツール実行権限を与えない
- LLM Provider は抽象化し、Ollama へ差し替え可能にする
- 将来的に高品質クラウドLLMへ差し替えられる構造を保つ

## 採用技術
- Python
- FastAPI
- SQLite
- Crawl4AI
- Ollama(OpenAI互換API)
- Docker Compose
- Pydantic
- pytest

## 必須機能
1. 対象サイト定義
2. 定期巡回
3. 記事抽出
4. 重複排除
5. 要約
6. 分類
7. 重要度判定
8. ユーザー反応保存
9. 推薦順位更新
10. Web UIでの一覧表示
11. フィードバック登録
12. JSON schema validation
13. コンテナ分離

## 非機能要件
- 未信頼入力を扱う層と本丸を分離する
- 収集コンテナは権限最小
- 判定コンテナは外部アクション不可
- DB保存前にバリデーションを通す
- ログを残す
- 例外時も全体停止しにくい構造にする

## まずやること
1. リポジトリ構成を提案
2. PROJECT.md を作業の進行に合わせて更新
3. AGENTS.md を前提に作業規約へ従う
4. MVPスコープを定義
5. Docker Compose 構成を作る
6. FastAPI の骨組みを作る
7. SQLite スキーマを定義
8. Crawl4AI を使った収集層を実装
9. Ollama Provider を実装
10. ランカー・要約器を実装
11. シンプルなUIを実装
12. pytest を整備

## 出力方針
- まず全体構成を示す
- 次にファイル単位で実装する
- 重要判断は PROJECT.md に記録する
- セキュリティ上の懸念は都度明示する
- 破壊的変更前には理由を書く
