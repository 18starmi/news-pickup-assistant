# アーキテクチャ要約

## 全体像
- crawler: 対象サイトから未信頼データを取得
- extractor: 生HTMLや生レスポンスから構造化データを抽出
- ranker: Ollama を使って要約・分類・重要度判定
- api: 安全な構造化データを返す
- ui: 自分向けニュースサイト風UI
- db: 永続保存
- scheduler: 定期巡回ジョブ管理
- notifier: 将来的な通知専用コンポーネント

## セキュリティ原則
- crawler は最も汚れやすいが、最も権限を弱くする
- extractor は未信頼入力を構造化するだけ
- ranker は判断するだけで、外部通知やシェル実行をしない
- api / ui は加工済みデータのみ扱う
- db 保存前に schema validation を行う

## 最終ユーザー体験
- ニュースサイト風の一覧画面
- タイトル、出典、日時、3行要約、重要理由、タグを表示
- 深掘りたい / 役に立った / 興味ない の反応ボタン
- 本当に重要なものだけ通知
