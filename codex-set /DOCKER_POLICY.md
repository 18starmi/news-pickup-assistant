# Docker / Compose 方針

## 基本原則
- 機能ごとではなく、信頼レベルと権限ごとに分離する
- 未信頼入力を受ける層と、本丸を分離する
- できるだけ最小権限にする

## サービス責務
### crawler
- 特定サイトを巡回するだけ
- 外部通信あり
- ホストの重要ファイルへアクセスしない
- シークレットを極力持たない

### extractor
- 生データから title / main_text / published_at / source_url などを抽出
- 外部通信なし

### ranker
- Ollama に接続して要約・分類・重要度判定を行う
- 外部通知なし
- シェル実行なし

### api
- DBから安全なデータのみ返す

### ui
- api 経由で表示するだけ

### db
- 永続保存専用

### notifier
- 将来的に DB の加工済みデータだけを読み、通知する
- ranker から直接通知させない
